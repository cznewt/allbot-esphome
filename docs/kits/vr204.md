# VR204 — 2-legged ALLBOT

Upstream category: [manuals.whadda.com/category.php?id=84](https://manuals.whadda.com/category.php?id=84) · Building instructions: [article 389](https://manuals.whadda.com/article.php?id=389).

The smallest ALLBOT — a biped with one hip and one ankle servo per leg. Great starter kit: four servos, predictable geometry, and short gait sequences you can tune by eye.

## Kit at a glance

- 4 × 9 g servos (hip left/right, ankle left/right)
- Backbone, 2 servo brackets, 2 legs with feet
- Original electronics: Arduino UNO + VRSSM servo shield + VRBS1 battery shield

## Mechanical build

Follow the Whadda manual for the mechanical parts verbatim. Nothing the ESPHome port changes affects the chassis.

As with every ALLBOT, the single most important step is centring each servo at 90° **before** clipping the leg on. An off-centre horn offsets the whole leg by however many spline teeth you missed, and every gait inherits the error.

## Servo map used by `example/vr204.yaml`

| Joint | Servo id | PCA9685 channel | `flipped` | `initial_angle` |
| --- | --- | --- | --- | --- |
| Left hip    | `hip_l`   | 0 | yes | 90 |
| Right hip   | `hip_r`   | 1 | no  | 90 |
| Right ankle | `ankle_r` | 2 | no  | 90 |
| Left ankle  | `ankle_l` | 3 | yes | 90 |

The `flipped` column matches the original `BOT.attach(..., flipped, 0)` flag in `VR204.ino`. Note the asymmetric order (the Velleman enum is hip_l → hip_r → ankle_r → ankle_l, not `_r` then `_l`) — it's copied faithfully so existing calibration notes transfer.

## Gaits exposed as Home Assistant actions

`walk_forward`, `walk_backward`, `walk_left`, `walk_right`, `look_left`, `look_right`, `scared`, `chirp`, `stop`.

`walk_forward` / `walk_backward` are longer than the lateral gaits: they splay the hips to 130° / 40°, then swing the ankles through four positions per stride. The lateral `walk_left` / `walk_right` only use the ankles.

## Expected resting pose

After boot, both hips sit at 90° (legs pointing straight down), both ankles at 90° (feet flat). The robot should stand upright without effort. If it leans:

- **Forward/back lean:** an ankle is off-centre — re-seat at 90°, or dial in `offset:` in YAML.
- **Twist:** a hip spline is misaligned — the `flipped` flag in my table should match the original VR204.ino, so if it's wrong the servo horn is simply mounted wrong.

## Typical first-run sequence

1. Flash `example/vr204.yaml` with `secrets.yaml` filled in.
2. Hold or prop the robot up so it can't tip over.
3. Call `chirp` from HA — verifies the buzzer and API.
4. Call `look_left` then `look_right` — low-risk, only the hips move.
5. Set it on the floor, call `walk_forward`.

VR204 gait tuning is mostly about the hip splay angles during `walk_forward` / `walk_backward` (130°/40°). If the robot is too short-stepped or tips over, adjust those in YAML — the rest of the gait follows.
