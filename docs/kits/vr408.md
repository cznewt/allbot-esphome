## Kit at a glance

- 8 ├Ś 9 g servos (two per leg: hip + knee)
- Backbone parts A / B / C, four servo brackets, four swing-arm/foot assemblies
- Original electronics: Arduino UNO + VRSSM servo shield + VRBS1 battery shield

## Mechanical build

Follow the Whadda manual verbatim for the mechanical parts ŌĆö backbone, brackets, swing arms, feet. The ESPHome port only changes the controller stack, not the chassis.

Key points from the manual that are easy to miss:

- **Center every servo before clipping a swing arm on.** The manual flags this as critical; an off-center spline rotates the resting pose by however many teeth you were off.
- **Keep the joint nuts finger-tight** so the arm pivots freely.
- **Align the signal wire (yellow) to the correct side of the shield headers.** With our PCA9685, the rule is the same: yellow = signal, plugged into the row matching each channel number.

## Gaits exposed as Home Assistant actions

`walk_forward`, `walk_backward`, `walk_left`, `walk_right`, `turn_left`, `turn_right`, `lean_{forward,backward,left,right}`, `wave_{front,rear}_{left,right}`, `scared`, `chirp`, `stop`.

Every gait is a single YAML action; call it repeatedly with `repeat:` in a script / automation if you want N steps.

## Expected resting pose

After boot, all hips should sit at ~45┬░ (legs pointing forward-out), all knees at ~45┬░ (feet flat on the ground). If a leg lifts or collapses:

- **Collapsed / splayed:** knee on that leg is probably mis-calibrated or the `flipped` flag is wrong. Re-centre the servo, or flip the flag in YAML.
- **Rotated forward/back:** hip spline is off by one tooth. Pop the horn, re-seat at 45┬░.
- **Foot rolls outward:** rare, usually indicates a bent swing arm from a fall.

## Typical first-run sequence

1. Flash `example/vr408.yaml` with your Wi-Fi in `secrets.yaml`.
2. Put the robot on a book so the feet don't touch the ground.
3. From HA, call the `chirp` action ŌĆö verifies the buzzer and API.
4. Call `scared` ŌĆö a low-risk gait that only shakes knees/hips in place.
5. Put the robot on the floor, call `walk_forward`.

If step 5 looks wrong, it's almost always servo calibration, not YAML timing.
